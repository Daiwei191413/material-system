⚡ BOM 整理神器 快速上手

一款浏览器端的 BOM 对比 + 自动补齐工具，用来把"客户/开发给的乱格式 BOM"统一整理成公司标准格式， 不用装任何软件，打开即用：
🔗 https://techphant-bom-tools.pages.dev

① 它能做什么

  • BOM 对比：上传两份 BOM，自动找出差异（字段变更/独有物料）并标红
  • 一键整理：按 A 模板的标准格式整理 B，自动补齐物料名称、参数描述、品牌等字段
  • 物料库沉淀：导入的物料自动存浏览器，下次打开免重复导入；A BOM 上传后新料也会自动进库
  • 立创联动：带立创编号的物料一键拉真实参数（厂家/封装/规格）
  • 输出 Excel：整理后直接下载，可提交客户或归档

② 基本操作（3 步走）

1

导入物料库（只需第一次）

进入 📦 物料编码 页 → 点 📥 导入物料库 Excel → 选择公司的 器件BOM库.xlsx。

导入后数据持久化在浏览器里，下次打开不用再导。页面会显示"已加载 XXX 条物料"。

2

上传两份 BOM

切到 🔍 BOM对比 页，分别上传：

· BOM A（基准版本）：作为输出标准的模板，通常是公司内部标准 BOM

· BOM B（对比版本）：需要整理/核对的来料 BOM

支持 .xlsx / .xls / .csv。A 上传后里面的新物料会自动增量入库（按型号+封装去重，已有不覆盖）。

3

对比 + 整理 + 导出

点 🔍 开始对比 查看差异总览和明细；确认无误后点 ✨ 按A格式整理B， 系统会用物料库 + 立创数据自动补齐字段，输出一份 标准格式 Excel，文件名会自动清洗（去掉"立创BOM"等后缀）。

④ 三级匹配策略

物料从 A 整理到输出时，系统按"精度优先"逐级尝试匹配物料库：
等级依据说明
L1立创编号精确匹配最高优先级，立创编号一致直接命中
L2型号 + 封装 复合键没立创号时用"型号+封装"双字段配对
L3规范化型号容差型号前缀/后缀差异容错，仅在封装兼容时生效
未命中—系统不会编造，保留原始描述让你人工补齐或走"批量一键补齐"拉立创

⑤ 参数描述规则（精度优先·不编造）

  • 电容/电阻的 精度、电压、介质、功率 必须来自 BOM 库 或 立创实际数据，缺任一字段 → 保留原始描述，不兜底猜
  • 容值/阻值可从型号回退（比如 10K、100NF 这种型号自带信息）
  • 功率只做单位换算（62.5mW → 1/16W），不按封装"默认假设"
  • 输出格式示例：MLCC 1.2pF ±0.1pF 50V NPO SMD 0402

⑥ 常见问题

问题解决办法
页面看到的版本号不是 V1.0.0 按 Ctrl + F5 强刷一次（浏览器缓存）
点「一键补齐」报 "Failed to fetch" 刷新页面重试；持续失败联系 David.hao
清空数据后物料库空了 之前导出过备份就用 导入物料库 Excel 恢复；没备份只能重新导
某物料一直未命中 先看"匹配详情"里缺什么字段；有立创编号的点「批量一键补齐」；没立创编号的去立创搜一下手动贴参数
整理后某字段是空的 说明 BOM 库 + 立创都没提供这个字段的数据，工具不编造，请人工核对补齐
小建议：物料库用得越多越准。第一次用可以先把公司最新的 器件BOM库.xlsx 导一次， 之后每处理一份新 A BOM 都会自动沉淀新物料，下次匹配命中率会越来越高。
